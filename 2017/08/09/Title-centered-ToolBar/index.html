自定义标题居中的ToolBar

发表于 2017-08-09 | 分类于 Android ， UI | 阅读次数

系统提供的ToolBar功能完善，使用方便。但现在很多APP都要求标题居中显示，ToolBar的样式不符合设计。以前是自己定制一个TitleView，但现在有ToolBar这么好的控件，所以试着在ToolBar的基础上自定义一个。通过阅读ToolBar的源码，成功达到了目标。ToolBar继承自ViewGroup,所以是可以往里面加入View的，同时有一个内部类LayoutParams，支持Gravity属性，所以我们使用LayoutParams设置加入ToolBar的View的属性。最终自定义了一个TitleBar，可以标题居中，可以往左边、中间、右边添加自定义View。

实现原理

ToolBar继承自ViewGroup,所以是可以往里面加入View的，同时有一个内部类LayoutParams，支持Gravity属性，所以我们使用LayoutParams设置加入ToolBar的View的属性。

居中的自定义View

在实际开发总有其他一些情况，在ToolBar中间不是一个标题，这时我们可以将自定义View放到ToolBar中间。addCenterView()方法将View加到ToolBar中间：

我们主要看下addCustomView()方法，有两个参数：自定义view和要放在ToolBar中的位置gravity。利用自定义view的LayoutParams属性vlp，生产我们添加view时的LayoutParams属性lp。代码调用如下：

  1. 拿到view的LayoutParams属性vlp
  2. 首先判断如果vlp为null，就调用generateDefaultLayoutParams()创建默认lp。
  3. 然后调用ToolBar的方法checkLayoutParams检测vlp不为空并且是LayoutParams，就利用vlp创建lp。generateLayoutParams()会根据vlp的类型创建lp。
  4. 前面的判断都不满足时就将vlp强转为lp。

可以看到这里的关键就是创建加入ToolBar的view的LayoutParams。

左边或右边的自定义的View

同样，如果想将自定义view加到ToolBar的左边或右边，只需指定Gravity就行。addLeftView()和addRightView()可以将多个view加到左边、或右边。

左边的View

往ToolBar左边添加view时，由于ToolBar默认会有一些间距，需要调整一下。首先看下ToolBar的默认属性：

==contentInsetStart==是左边的间距，默认是16dp。当我们往左边加入view时，使用 mTitleBar.setContentInsetsRelative(0, mTitleBar.getCurrentContentInsetEnd())将该间距设为0。

==contentInsetStartWithNavigation==是NavigationIcon与其右边的view距离，默认是72dp。当ToolBar已经设置NavigationIcon时，往左边加入view会放在ToolBar右边，使用mTitleBar.setContentInsetStartWithNavigation(0)方法设置间距为0。
